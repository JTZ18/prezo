The Problem

Most detections are redundant — same person, similar angle, similar pose. Manual annotation of detailed attributes (clothing, accessories, actions) is prohibitively expensive. We need automation that's both fast AND accurate.

Our Solution

A multi-stage pipeline that processes video at 500+ FPS, intelligently deduplicates to 5 diverse crops per identity, generates forensic-quality captions, and validates through dual-layer verification.

Tracking with CLIP-ReID Integration

NvDCF tracker uses a weighted combination of three signals for data association:

CLIP-ReID Embeddings — Dominant signal for re-identification 50%
50%
Mahalanobis Distance — Motion prediction via Kalman filter 30%
30%
Size Similarity — Bounding box dimension matching 20%
20%

Occlusion Handling

512-dimensional CLIP embeddings extracted every 5 frames. We maintain 5 best embeddings per track for robust re-identification. Shadow tracks persist for 30 frames without detection. Hungarian algorithm assigns with 0.6 threshold.

Full-Body Validation (Critical)

We run YOLOv8n-pose and check that at least 2 of 6 lower-body keypoints (hips, knees, ankles) are visible with confidence > 0.5. Partial-body crops create inconsistent captions — one crop might describe full clothing, another just the torso.

Why Pose-Based Deduplication?

100+

Crops per track

→

K-Means

34-dim pose vectors

→

5

Diverse crops

Three Key Advantages

  • Zero additional inference cost — Keypoints already computed during full-body filtering
  • Semantic meaning — Different poses correlate with different camera angles
  • Robustness — Pose features invariant to minor appearance changes

Algorithm

  • 17 COCO keypoints × 2 coords = 34-dim feature vector
  • K-Means with k = target crop count (default 5)
  • Select highest quality score from each cluster
  • Result: 5 crops with diverse viewing angles
9

Quality Score Calculation

The quality score determines which crop gets selected from each pose cluster:

Sharpness — Normalized Laplacian variance (capped at 1.0 for variance ≥ 500) 50%
50%
Edge Penalty — 15% per edge cut at frame boundary (max 60%) 30%
30%
Aspect Ratio — Ideal range 0.3-0.7 (typical standing person proportions) 20%
20%
# Quality score formula
quality_score = (
    sharpness_score * 0.5 +
    (1.0 - edge_penalty) * 0.3 +
    aspect_score * 0.2
)

Dual-Layer Validation

Validation Architecture

Generated captions can hallucinate. We validate through two parallel mechanisms:

🧠 Semantic Validation 70%

Ministral-8B-Instruct acts as judge

  • Accuracy: Does description match visible content?
  • Completeness: Are key features mentioned?
  • Consistency: Any internal contradictions?
  • Hallucination: Mentions items not visible?

👀 Visual Grounding 30%

SAM3 verifies objects exist

  • Extract critical nouns from caption
  • Query SAM to find each object
  • Calculate verification pass rate
  • Threshold: 80% for pass
# Combined validation score
overall_score = (semantic.score * 0.7) + (grounding.pass_rate * 100 * 0.3)

Key Design Decisions

1

Process Every Frame

Don't skip frames. Let deduplication handle redundancy intelligently.

2

Pose-Based Deduplication

More semantically meaningful than pixel similarity, zero additional cost.

3

Two-Stage GPU Execution

Practical solution for memory constraints on single GPU.

4

Guided Captioning

Maintain consistency across crops of same identity.

5

Dual-Layer Validation

Semantic judgment + visual grounding catches different failure modes.

6

Quality-First Selection

From each pose cluster, always pick the sharpest crop.
